# V15 更新记录

发布日期：2026-07-24

## API 设置

- 新增仅监听 `127.0.0.1:51415` 的 Local API。
- 支持环境查询、创建、更新、启动、停止、删除、代理检测、会话查询和 OpenAPI 3.1 文档。
- 启动接口返回 `debuggerAddress`、`browserURL` 和 `browserWSEndpoint`，可连接 Selenium 或 Puppeteer。
- 环境通过 API 创建或更新后，客户端列表会实时同步。

## 安全措施

- API Key 强制验证，不提供关闭验证的选项。
- 支持 `X-API-Key` 与 Bearer Token。
- 校验 Host 与 Origin，不开放 CORS；限制请求体为 1 MB、每分钟 600 次请求。
- 查询结果隐藏代理密码、Cookie、代理刷新 URL、浏览器可执行路径和 Profile 路径。
- API Key 随机生成并存放在当前 Windows 用户的应用数据目录；可在客户端一键重置。

## MCP Server

- 新增无外部运行时依赖的 stdio MCP Server。
- 提供环境列表、详情、创建、更新、启动、停止、代理检查和会话查询共 8 个工具。
- 提供 `xiaohei://local-api/openapi` 资源。
- MCP 不暴露删除工具，也不会把 API Key写进客户端配置。

## Skill 技能包

- 新增 `skills/xiaohei-browser`。
- 包含 MCP 优先、安全操作规则、Local API、Selenium 与 Puppeteer 参考。
- 适用于支持 Skills/MCP 配置的 Claude Code、Codex、Cursor、OpenCode、Gemini GL、OpenClaw 和 Hermes agent；具体安装目录以各工具文档为准。

## 客户端

- 左侧新增“API & MCP”入口。
- 支持接口启停、状态显示、地址与 Key 复制、Key 重置、MCP 配置复制、Skill 目录打开。
- 新增 Selenium/Puppeteer 示例切换和接口清单。
